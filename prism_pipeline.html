Training → Inference Pipeline

PRISM 从训练到推理的完整流程

两个阶段：校准 + EIG 回归训练 把噪声信道参数和路由器学进来； 序贯贝叶斯推理 在每条新轨迹上更新后验、按 EIG 路由、按 conformal 阈值停止。 PRISM 把 LLM 的四条输出当作四条独立的贝叶斯证据信道： stage1 软参考一致性、stage2 标签、specialist 三件套、（可选）review。

Phase 1 · Training

训练流程

把"噪声信道 + EIG 路由器 + conformal 阈值"三件事都学进来

1

收集证据轨迹 ~ 一次 forward pass 的数据集

在一批训练轨迹上跑 --pipeline prism（或 legacy）以收集每步的 stage2 标签 logits 和 specialist tool trace。把所有 StepPrediction 持久化到 artifacts/predictions/*.jsonl。 这一步唯一的产物是原始证据，不是任何决策。 

$ mathchecker run \
    --dataset big-bench-mistake --model gpt-4o-mini \
    --stage2-tools triad --pipeline prism
2

校准似然函数 isotonic regression on hold-out

对每条证据通道 $p_i(V^{(i)} \mid \tau)$ 拟合 sensitivity 参数：在一个 hold-out 子集上， 用 isotonic regression 把 specialist 报告的 raw confidence 校准到经验出错率上。 每个 specialist 一个 scalar sensitivity，stage2 一个，review 一个。

校准目标
$$\widehat{s}_i = \arg\min_{s \in [0,1]} \sum_{(V^{(i)},\,\tau^\star)} -\log p_i(V^{(i)} \mid \tau^\star;\, s)$$

产出：每条信道的标定 sensitivity，存到 artifacts/prism_likelihoods.json。

3

生成 EIG 回归标签 closed-form, deterministic

在每个 (轨迹, 步 t) 上重放后验 $\pi_t$（用步 t 及之前的证据更新）。然后对每个候选 specialist $c$ 用闭式公式直接算 $\mathrm{EIG}^\star(\pi_t, c) = H(\pi_t) - \mathbb{E}_{V}[H(\pi_t \mid V)]$。 这就是 router 训练的目标值——无 counterfactual、无 RL、deterministic。 

$ python scripts/train_prism_router.py \
    --predictions-path artifacts/predictions/big-bench-mistake__gpt-4o-mini.jsonl \
    --export-path artifacts/prism_router/train.jsonl

每条样本含 posterior_probs、posterior_entropy_nats、 principle_labels，以及 dense real-valued specialist_eig_target（每个 specialist 一个标量）。

4

训练小模型回归 EIG Qwen3-0.6B + LoRA

用一个轻量 base（Qwen3-0.6B）配 LoRA adapter，把上一步生成的 (context, EIG*) 样本拿来做 多目标回归。损失：

PRISM Router 训练损失
$$\mathcal{L}(\theta) = \mathbb{E}_{(\pi_t,\, x)}\Big[\, \big\| r_\theta(\pi_t, x) - \mathrm{EIG}^\star(\pi_t, x) \big\|^2 \,\Big]$$

输入 context：question + previous steps + current step + stage1 软参考 + 当前后验摘要 (entropy、max-mass、step-type)。输出：长度 $M$ 的实值向量（每个 specialist 的预期信息量）。

5

校准 conformal 阈值 $\alpha_t$ Vovk-Shafer e-process

在 hold-out 上跑完整 PRISM (early_stop=False)，记录每条轨迹的 $\max_\tau \pi_t(\tau)$ 序列。 对每个时刻 $t$ 选 $\alpha_t$ 为 $(1-\delta)$ 经验分位数，保证在 committed 子集上 $P(\hat\tau = \tau^{\text{true}} \mid \text{commit}) \geq 1 - \delta$。

阈值校准
$$\alpha_t = \text{Quantile}_{\,\delta}\left\{ \max_\tau \pi_t(\tau) \;:\; \text{trace} \in \mathcal{D}_{\text{cal}} \right\}$$

产出 artifacts/prism_conformal.json（schedule + delta + calibration_size）。

Phase 2 · Inference

推理流程

每条新轨迹按步序进行，所有决策围绕同一个 $\pi_t(\tau)$

0

初始化后验 $\pi_0(\tau)$

length_prior(num_steps, p_no_error=0.4)：tent-shaped 先验 over 步索引， 保留 0.4 给 $\tau = \infty$。observed_up_to = 0。

1

Stage1 + Stage2 LLM 调用 每步必跑的免费证据

复用 PedCoTPredictor._run_stage1 / _run_stage2。 Stage1 先跑，输出 {mathematical_concepts, key_analyses, calculations}——是对该步该计算什么的独立预测。 Stage2 接着跑（以 stage1 输出为 conditioning），产出三维 principle_labels， 并把 specialist 三件套挂在 function-call schema 里。

2a

Stage1 一致性信道 独立的数值校验

extract_stage1_inconsistency(stage1.calculations, current_step) 从两段文本里抽取 "$\text{lhs} = \text{rhs}$" 形式的数值等式：若同一 lhs 在 stage1 与 step 中给出不同 rhs，或 step 的 $\text{lhs}$ 在符号计算下与 step 自报的 rhs 不一致 → 硬 inconsistency。 这条信道独立于 stage2 标签，捕捉"stage1 已经预测过结果、step 实际算错"的情形。

Stage1 likelihood (sensitivity = 0.55, lower than stage2)
$$p_{S_1}(V_1 \mid \tau) = \begin{cases} \text{base} + s \cdot q_1 & \text{if } \tau = t \\ \text{base} + s \cdot 0.5 & \text{if } \tau < t \\ \text{base} + s \cdot (1-q_1) & \text{otherwise} \end{cases}\quad\text{其中}\;q_1 = \text{inconsistency\_strength}$$

$q_1 = 0$ 时似然均匀（no-op），不动后验。仅当能确凿匹配出数值矛盾时才非零。

2b

Stage2 标签信道 principle labels → likelihood

make_stage2_likelihood(step_index, principle_labels, sensitivity) 返回一个长度 $T+1$ 的 似然向量，按 $\pi_{t+1} \propto \pi_t \cdot p_S(S_t \mid \tau)$ 做 Bayes 更新。

Likelihood profile（stage2，"contradiction-found"）
$$p_S(S_t \mid \tau) = \begin{cases} \text{base} + s \cdot q & \text{if } \tau = t \\ \text{base} + s \cdot 0.5 & \text{if } \tau < t \\ \text{base} + s \cdot (1-q) & \text{if } \tau > t \text{ or } \tau = \infty \end{cases}$$
3

EIG 路由 greedy submodular

用 trained router $r_\theta(\pi_t, x)$ 预测每个 specialist 的 EIG，配合 greedy_select(budget, lam) 选出本步要触发的子集。 每个 specialist 都"按需调用"——低熵 / 高确定时直接不调，省钱在这里。

选择准则
$$S^\star = \arg\max_{S}\; r_\theta(S, \pi_t, x) - \lambda \sum_{i \in S} c_i$$
4

调用选中的 specialist LLM tool call

对每个 $i \in S^\star$ 实际发起 LLM tool call（已经 attached 在 stage2 prompt 里，所以是低成本 follow-up）。 解析返回的 evidence，提取 (hard_conflict_strength, valid_alternative_strength)。 对每个 emission 用 make_specialist_likelihood 做一次 Bayes 更新。

5

Φ-不变量投影 取代 review + fallback

融合 stage1 / stage2 / specialist 三条贝叶斯信道的有效"contradiction strength": $\;c_t = \max(q_{\text{stage1}},\, q_{\text{stage2}},\, q_{\text{spec,hard}}) \cdot (1 - \text{valid\_alt})$。 用过去所有观测步的 $c_{0:t}$ 计算 multiplier 并做 Bayes 投影。 valid-alternative 乘性抑制所有 contradiction 信号——这是 stage1 与 stage2 都能被 specialist "valid alternative" 证据下调的关键。

6

观测性门控 + Conformal 停止 decide & commit

两条条件必须同时成立才能 commit：

  • Observability gate：argmax 不能落在未观测步上。$\tau=\infty$ 要求 observed_up_to == num_steps。
  • Conformal threshold：$\max_\tau \pi_t(\tau) \geq \alpha_t$。

满足 → 输出 $\hat\tau = \arg\max \pi_t$；否则 → 进入下一步 / 末步 → abstain。
